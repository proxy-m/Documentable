Basics

Documentable uses a JSON file to read the custom configuration. You have three different options to specify it:

  • Creates a file called config.json in the same directory where Documentable will be executed.

  • Provide a custom path to a JSON file with the --conf option.

  • Do nothing. In that case, the default configuration file will be used.

kinds entry

This entry follow the format:

"kinds": [
    ...
]

Where each one of the elements is written as:

{
    "kind": "a kind name (see Kind)",
    "display-text": "text to be displayed in the menu buttom",
    "sort": 1,
    "categories": [
        "name": "sort name to be used by Documentable and link",
        "display-text": "text to be displayed in the menu buttom"
    ]
}

  • The value of kind must be the stringification of one element of Kind.

  • The values of name specified in categories has to be the same used in the metadata (see document-format).

  • sort option is only available for Kind::Language, to sort the documents using the categories in one column.

Additional entries

{
    "title-page": "this string will appear in the tab",
    "pod-root-path": "link to the documentation dir",
    "url-prefix": "some string"
}

  • pod-root-path: Base link of the directory containing the documentation. It will be used to enable the edit button at the beginning of evey page.

    For instance: "pod-root-path": "https://github.com/perl6/Documentable/tree/master/docs",.

  • url-prefix: This entry is very important if you are going to deploy your site to gh-pages. You may have realised that all URLs are relative to /. So, if you deploy the html directory to your gh-pages branch, none of your links will work because they will be pointing to the root of your .io domain. This happens because your repository page url is the concatenation of your .io domain with the name of the repository.

    To avoid this problems, you can specify the name of your repo as the value of url-prefix (without "/"). This string will be added to the beginning of every url generated by Documentable. Take a look at the configuration file used by the Documentable documentation to see an example.
